## 一、HTTP 请求头与 Token 认证

### 1.1 什么是 HTTP 请求头

HTTP 请求头（Headers）是客户端发送给服务器的附加信息，用于描述请求的属性和上下文。它们就像是信件的"信封"，告诉服务器关于请求的一些重要信息。

**常见的请求头**：

| 请求头 | 作用 | 示例 |
| :--- | :--- | :--- |
| `Content-Type` | 指定请求体的数据类型 | `application/json` |
| `Authorization` | 携带认证信息 | `Bearer token123` |
| `Accept` | 告诉服务器接受什么类型的响应 | `application/json` |
| `User-Agent` | 标识客户端类型 | `Mozilla/5.0` |
| `Cookie` | 携带 Cookie 信息 | `session=abc123` |

### 1.2 Token 认证机制

在现代 Web 应用中，Token 认证是最常用的身份验证方式之一。

**Token 认证流程**：

```
用户登录成功
    |
    v
服务器生成 Token（一串加密字符串）
    |
    v
服务器返回 Token 给客户端
    |
    v
客户端存储 Token（localStorage/cookie）
    |
    v
每次请求时，客户端在请求头中携带 Token
    |
    v
服务器验证 Token 有效性
    |
    v
验证通过，返回数据
```

### 1.3 Bearer Token 格式

`Bearer` 是一种认证方案，它告诉服务器："我持有一个令牌（Token），请用这个令牌来验证我的身份"。

**格式**：

```
Authorization: Bearer <token>
```

**示例**：

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### 1.4 代码解析

```javascript
// 获取存储在 localStorage 中的 token
const token = localStorage.getItem('token')

// 如果 token 存在，添加到请求头
if (token) {
  config.headers.Authorization = `Bearer ${token}`
}
```

**这段代码的作用**：

1. **获取 Token**：从 `localStorage` 中读取之前存储的 Token
2. **检查 Token**：判断 Token 是否存在
3. **添加到请求头**：如果存在，将 Token 添加到 `Authorization` 请求头中

**`config.headers` 的含义**：

`config` 是 axios 请求的配置对象，`config.headers` 是请求头的集合。通过修改 `config.headers.Authorization`，我们可以在发送请求时自动带上认证信息。

**请求头的最终样子**：

当请求发送时，HTTP 请求头会包含：

```
Content-Type: application/json;charset=utf-8
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

## 二、window.location 对象详解

### 2.1 什么是 window.location

`window.location` 是浏览器提供的一个内置对象，用于获取或设置当前页面的 URL 信息。它就像是浏览器地址栏的编程接口。

### 2.2 window.location 的常用属性

| 属性 | 作用 | 示例值 |
| :--- | :--- | :--- |
| `href` | 获取或设置完整的 URL | `https://example.com/login` |
| `protocol` | 获取协议（http/https） | `https:` |
| `hostname` | 获取主机名 | `example.com` |
| `pathname` | 获取路径部分 | `/login` |
| `search` | 获取查询参数 | `?id=123` |
| `hash` | 获取锚点（# 后面的部分） | `#section1` |

### 2.3 window.location.href 的作用

`window.location.href` 是最常用的属性，它有两个用途：

**用途一：获取当前页面的完整 URL**

```javascript
// 获取当前页面的 URL
const currentUrl = window.location.href
console.log(currentUrl) // 输出: "https://example.com/home"
```

**用途二：跳转到新页面**

```javascript
// 跳转到登录页
window.location.href = '/login'

// 跳转到外部网站
window.location.href = 'https://google.com'
```

**这就是实现页面跳转的核心代码！**

### 2.4 页面跳转的场景

在我们的 `request.js` 中，`window.location.href = '/login'` 出现在两个地方：

```javascript
// 场景一：业务错误码 401（token 失效）
if (res.code === 401) {
  localStorage.removeItem('token') // 清除无效的 token
  window.location.href = '/login'  // 跳转到登录页
}

// 场景二：HTTP 状态码 401（未授权）
case 401:
  message = '未授权，请重新登录'
  localStorage.removeItem('token')
  window.location.href = '/login'
  break
```

**为什么要这样做？**

当用户的 Token 失效或过期时，服务器会返回 401 状态码。此时：

1. **清除旧 Token**：从 `localStorage` 中删除无效的 Token
2. **跳转到登录页**：让用户重新登录获取新的 Token

### 2.5 相关方法对比

除了 `href`，`window.location` 还有其他跳转方法：

| 方法 | 作用 | 是否可后退 |
| :--- | :--- | :--- |
| `location.href = 'url'` | 跳转到新页面，添加历史记录 | 是 |
| `location.replace('url')` | 跳转到新页面，替换当前历史记录 | 否 |
| `location.reload()` | 重新加载当前页面 | - |

**使用场景**：

- 使用 `href`：普通页面跳转（登录成功后跳转首页）
- 使用 `replace`：不希望用户回退的场景（登录页跳转后不希望返回登录页）
- 使用 `reload`：刷新当前页面

## 三、localStorage 本地存储

### 3.1 什么是 localStorage

`localStorage` 是浏览器提供的本地存储机制，用于在用户的浏览器中存储键值对数据。

**特点**：

| 特性 | 说明 |
| :--- | :--- |
| 存储大小 | 约 5MB |
| 生命周期 | 永久存储，除非手动清除 |
| 作用域 | 同一域名下的所有页面共享 |
| 数据类型 | 只能存储字符串 |

### 3.2 常用方法

```javascript
// 存储数据
localStorage.setItem('token', 'abc123')

// 获取数据
const token = localStorage.getItem('token') // 返回 "abc123"

// 删除数据
localStorage.removeItem('token')

// 清空所有数据
localStorage.clear()
```

### 3.3 Token 的存储流程

```
用户登录成功
    |
    v
服务器返回 Token
    |
    v
localStorage.setItem('token', token)
    |
    v
后续请求时：localStorage.getItem('token')
    |
    v
添加到请求头 Authorization
```

### 3.4 安全性注意事项

**localStorage 的优缺点**：

| 优点 | 缺点 |
| :--- | :--- |
| 存储容量大 | 只能存储字符串 |
| 持久化存储 | 可能被 XSS 攻击窃取 |
| 跨页面共享 | 不支持跨域 |

**安全建议**：

1. **敏感信息加密**：不要直接存储密码等敏感信息
2. **使用 HTTPS**：防止数据在传输过程中被窃取
3. **设置过期时间**：在业务层面为 Token 设置过期时间
4. **XSS 防护**：对用户输入进行过滤和转义

## 四、实际应用场景

### 4.1 用户登录流程

```javascript
// 登录成功后存储 Token
async function login(username, password) {
  const result = await api.post('/api/auth/login', {
    username,
    password
  })
  
  // 将 Token 存储到 localStorage
  localStorage.setItem('token', result.token)
  
  // 跳转到首页
  window.location.href = '/home'
}
```

### 4.2 请求拦截器自动添加 Token

```javascript
request.interceptors.request.use(
  config => {
    // 每次请求时自动获取 Token
    const token = localStorage.getItem('token')
    
    // 如果有 Token，添加到请求头
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }
    
    return config
  },
  error => {
    return Promise.reject(error)
  }
)
```

### 4.3 响应拦截器处理 Token 失效

```javascript
request.interceptors.response.use(
  response => {
    const res = response.data
    
    // 业务错误码 401：Token 失效
    if (res.code === 401) {
      localStorage.removeItem('token') // 清除无效 Token
      window.location.href = '/login'  // 跳转到登录页
    }
    
    return res
  },
  error => {
    // HTTP 状态码 401：未授权
    if (error.response?.status === 401) {
      localStorage.removeItem('token')
      window.location.href = '/login'
    }
    
    return Promise.reject(error)
  }
)
```

## 六、总结

### 6.1 核心概念回顾

| 概念 | 作用 | 关键代码 |
| :--- | :--- | :--- |
| **请求头** | 携带请求附加信息 | `config.headers.Authorization` |
| **Bearer Token** | 身份认证方式 | `Bearer ${token}` |
| **localStorage** | 本地存储 Token | `localStorage.setItem('token', token)` |
| **window.location.href** | 页面跳转 | `window.location.href = '/login'` |
| **拦截器** | 统一处理请求/响应 | `interceptors.request.use()` |

### 6.2 安全要点

1. **Token 要保密**：不要在控制台打印 Token
2. **使用 HTTPS**：防止 Token 被窃取
3. **设置合理过期时间**：避免 Token 长期有效
4. **Token 失效后及时清理**：避免使用无效 Token

### 6.3 常见问题

**Q1：为什么 Token 会失效？**

A：Token 失效通常有以下原因：
- Token 过期（服务端设置了有效期）
- 用户修改了密码
- 用户被管理员注销
- Token 格式错误

**Q2：为什么要清除 localStorage 中的 Token？**

A：当 Token 失效后，如果不清除：
- 后续请求会继续携带无效 Token
- 服务器会持续返回 401 错误
- 用户会陷入无限重定向循环

**Q3：window.location.href 和 router.push() 有什么区别？**

A：
- `window.location.href`：强制刷新页面，重新加载整个应用
- `router.push()`：Vue Router 的方法，只更新视图，不刷新页面

在 Token 失效的场景中，使用 `window.location.href` 可以确保完全重新加载应用，清除可能存在的缓存状态。
