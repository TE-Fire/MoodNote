## 二、Filter（过滤器）与 Interceptor（拦截器）对比

### 2.1 核心区别一览

| 特性 | Filter | Interceptor |
|------|--------|-------------|
| **所属框架** | Servlet API（Tomcat/Jetty） | Spring MVC |
| **配置位置** | `web.xml` 或 `@WebFilter` | `WebMvcConfigurer` |
| **执行顺序** | FilterChain 顺序 | 拦截器列表顺序 |
| **作用范围** | 所有请求（包括静态资源） | Spring MVC 映射的请求 |
| **API 层级** | 底层 Servlet 级别 | 更高层次的 Spring 封装 |
| **可以获取** | ServletRequest、ServletResponse | HandlerMethod（包含方法签名） |
| **典型用途** | CORS、字符编码、CSRF | 登录验证、权限校验、日志 |

### 2.2 Filter 工作原理

```java
/**
 * Filter 是 Servlet 规范的一部分，位于 Tomcat 容器层
 * 所有请求（包括静态资源）都会先经过 Filter
 *
 * 执行顺序：Filter1.doFilter() → Filter2.doFilter() → Servlet
 *          ←───────────────────  ←─────────────────────
 */

@WebFilter(urlPatterns = "/*")
public class AuthenticationFilter implements Filter {

    @Override
    public void init(FilterConfig filterConfig) throws ServletException {
        // 初始化时调用，可获取 Filter 配置参数
        System.out.println("AuthenticationFilter 初始化");
    }

    @Override
    public void doFilter(ServletRequest request, ServletResponse response,
                         FilterChain chain) throws IOException, ServletException {

        HttpServletRequest httpRequest = (HttpServletRequest) request;
        HttpServletResponse httpResponse = (HttpServletResponse) response;

        // ========== 拦截逻辑（请求到达前） ==========

        String uri = httpRequest.getRequestURI();
        System.out.println("请求路径: " + uri);

        // 放行请求，继续执行后续 Filter 或 Servlet
        chain.doFilter(request, response);

        // ========== 响应逻辑（响应返回前） ==========

        System.out.println("响应状态码: " + httpResponse.getStatus());
    }

    @Override
    public void destroy() {
        // 销毁时调用，用于资源清理
        System.out.println("AuthenticationFilter 销毁");
    }
}
```

### 2.3 Interceptor 工作原理

```java
/**
 * Interceptor 是 Spring MVC 的组件，位于 DispatcherServlet 之后
 * 只作用于 @RequestMapping 映射的请求
 *
 * 执行流程：
 * 1. preHandle(...)     → Controller 执行前（返回 true 才继续）
 * 2. Controller 执行    → 业务逻辑处理
 * 3. postHandle(...)    → Controller 执行后，视图渲染前
 * 4. afterCompletion()  → 整个请求完成后（无论成功失败）
 */

@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new AuthenticationInterceptor())
                .addPathPatterns("/**")                    // 拦截的路径
                .excludePathPatterns(                      // 排除的路径
                        "/api/auth/login",
                        "/api/auth/register",
                        "/api/auth/captcha",
                        "/api/auth/sendCode",
                        "/static/**",
                        "/error"
                );
    }
}
```

```java
/**
 * 认证拦截器示例
 * 用于验证 JWT Token 的有效性
 */
public class AuthenticationInterceptor implements HandlerInterceptor {

    @Autowired
    private JwtUtil jwtUtil;

    @Autowired
    private StringRedisTemplate redisTemplate;

    @Override
    public boolean preHandle(HttpServletRequest request,
                             HttpServletResponse response,
                             Object handler) throws Exception {

        // 1. 获取请求路径
        String uri = request.getRequestURI();

        // 2. 如果是预检请求（OPTIONS），直接放行
        if ("OPTIONS".equalsIgnoreCase(request.getMethod())) {
            return true;
        }

        // 3. 获取 Token（优先从 Header 获取）
        String token = request.getHeader("Authorization");

        if (token == null || token.isEmpty()) {
            // 也可以从 Cookie 获取
            Cookie[] cookies = request.getCookies();
            if (cookies != null) {
                for (Cookie cookie : cookies) {
                    if ("token".equals(cookie.getName())) {
                        token = cookie.getValue();
                        break;
                    }
                }
            }
        }

        // 4. 验证 Token
        if (token == null || token.isEmpty()) {
            response.setStatus(401);
            response.setContentType("application/json;charset=UTF-8");
            response.getWriter().write("{\"code\":401,\"message\":\"No token provided\"}");
            return false;
        }

        try {
            // 验证 Token 有效性
            Claims claims = jwtUtil.parseToken(token);

            // 检查 Token 是否在黑名单（用户登出时加入）
            String blackListKey = "blacklist:token:" + token;
            if (Boolean.TRUE.equals(redisTemplate.hasKey(blackListKey))) {
                response.setStatus(401);
                response.setContentType("application/json;charset=UTF-8");
                response.getWriter().write("{\"code\":401,\"message\":\"Token has been invalidated\"}");
                return false;
            }

            // 5. 将用户信息存入请求属性，供 Controller 使用
            request.setAttribute("userId", claims.get("userId"));
            request.setAttribute("username", claims.get("username"));

            return true;

        } catch (ExpiredJwtException e) {
            response.setStatus(401);
            response.setContentType("application/json;charset=UTF-8");
            response.getWriter().write("{\"code\":401,\"message\":\"Token expired\"}");
            return false;
        } catch (JwtException e) {
            response.setStatus(401);
            response.setContentType("application/json;charset=UTF-8");
            response.getWriter().write("{\"code\":401,\"message\":\"Invalid token\"}");
            return false;
        }
    }

    @Override
    public void postHandle(HttpServletRequest request,
                           HttpServletResponse response,
                           Object handler,
                           ModelAndView modelAndView) throws Exception {
        // Controller 执行后、视图渲染前执行
        // 可用于修改 ModelAndView
    }

    @Override
    public void afterCompletion(HttpServletRequest request,
                                HttpServletResponse response,
                                Object handler,
                                Exception ex) throws Exception {
        // 整个请求完成后执行，无论成功或失败
        // 可用于清理资源、记录日志等
        if (ex != null) {
            System.out.println("请求异常: " + ex.getMessage());
        }
    }
}
```

## 三、Token 存储方案：localStorage vs HTTP-only Cookie

### 3.1 方案对比

| 特性 | localStorage | HTTP-only Cookie |
|------|-------------|------------------|
| **JavaScript 访问** | ✅ 可读写 | ❌ 无法访问 |
| **XSS 攻击风险** | 高（JS 可直接读取） | 低（JS 无法读取） |
| **CSRF 攻击风险** | 低（需手动附加到 Header） | 高（浏览器自动发送） |
| **跨域携带** | 需手动配置 | 需设置 `withCredentials` |
| **过期管理** | 需前端自行处理 | 可设置 Max-Age |
| **移动端支持** | ✅ | ✅ |
| **推荐场景** | 单页面应用 | 需要高安全性场景 |

### 3.2 XSS 攻击原理（localStorage 风险）

```

│                         XSS 攻击示意图                                 │
```

### 3.3 HTTP-only Cookie 详解

#### 什么是 HTTP-only？

`HTTP-only` 是 Cookie 的一个安全属性，通过设置 `Set-Cookie` 响应头实现：

```
Set-Cookie: token=eyJhbGciOiJIUzI1NiJ9...; HttpOnly; Secure; SameSite=Lax
```

当设置 `HttpOnly` 标志后：
- **JavaScript 无法通过 `document.cookie` 访问该 Cookie**
- **浏览器仍然会在后续请求中自动携带该 Cookie**
- **XSS 攻击者无法通过 JavaScript 读取 Token**

#### HTTP-only 的工作原理

```
┌────────────────────────────────────────────────────────────────────────┐
│                      HTTP-only Cookie 工作流程                          │
└────────────────────────────────────────────────────────────────────────┘

1. 服务器登录成功后，设置 HTTP-only Cookie：
   HTTP/1.1 200 OK
   Set-Cookie: token=eyJhbGciOiJIUzI1NiJ9...; HttpOnly; Secure; Path=/

2. 浏览器存储该 Cookie（标记为 HTTP-only）

3. 后续请求浏览器自动携带：
   GET /api/user HTTP/1.1
   Host: example.com
   Cookie: token=eyJhbGciOiJIUzI1NiJ9...

4. JavaScript 无法读取：
   console.log(document.cookie);  // "token=eyJhbGciOiJIUzI1NiJ9..." 不会出现！
```

### 3.4 CSRF 攻击原理（HTTP-only Cookie 风险）

```
┌────────────────────────────────────────────────────────────────────────┐
│                         CSRF 攻击示意图                                 │
└────────────────────────────────────────────────────────────────────────┘

攻击者诱导已登录用户访问恶意页面：

1. 恶意页面包含自动提交的表单：
   <html>
     <body>
       <form action="https://bank.com/transfer" method="POST" id="csrf-form">
         <input name="to" value="attacker" />
         <input name="amount" value="10000" />
       </form>
       <script>document.getElementById('csrf-form').submit();</script>
     </body>
   </html>

2. 用户已登录银行网站，Cookie 中存有有效 Token

3. 浏览器自动携带 Cookie 发送请求：
   POST /transfer HTTP/1.1
   Host: bank.com
   Cookie: token=eyJhbGciOiJIUzI1NiJ9...  ◀── 浏览器自动携带！
   Content-Type: application/x-www-form-urlencoded

   to=attacker&amount=10000  ◀── 转账成功（后端无法区分是用户还是攻击者）

4. 后端认为请求来自合法用户，完成转账
```

### 3.5 最佳安全实践：HTTP-only + CSRF Token

#### 方案设计

```
┌────────────────────────────────────────────────────────────────────────┐
│                        双重安全防护方案                                 │
└────────────────────────────────────────────────────────────────────────┘

┌──────────────────┐         ┌──────────────────┐         ┌──────────────────┐
│   HTTP-only      │  防护   │    CSRF Token    │  防护   │     HTTPS       │
│   Cookie         │   XSS   │    双重验证      │  CSRF   │    加密传输      │
└──────────────────┘         └──────────────────┘         └──────────────────┘
        │                            │                            │
        ▼                            ▼                            ▼
  Token 存储在              请求体/Header 中携带              防止中间人攻击
  Cookie 中                 CSRF Token                        防止抓包
  JS 无法读取               后端验证
```

#### 后端实现：JWT + HTTP-only Cookie

```java
/**
 * 登录成功后设置 HTTP-only Cookie
 */
@PostMapping("/login")
public Result<LoginVO> login(@RequestBody LoginDTO loginDTO, HttpServletResponse response) {

    // 1. 验证用户凭证...

    // 2. 生成 Token
    Map<String, Object> claims = new HashMap<>();
    claims.put("userId", user.getId());
    claims.put("username", user.getUsername());
    String token = jwtUtil.createToken(claims);

    // 3. 生成 CSRF Token
    String csrfToken = UUID.randomUUID().toString();
    // 存入 Redis，设置过期时间
    redisTemplate.opsForValue().set("csrf:" + csrfToken, user.getId().toString(),
            30, TimeUnit.MINUTES);

    // 4. 设置 HTTP-only Cookie
    Cookie tokenCookie = new Cookie("token", token);
    tokenCookie.setHttpOnly(true);          // HTTP-only，防止 XSS 读取
    tokenCookie.setSecure(true);            // 仅 HTTPS 传输
    tokenCookie.setPath("/");               // 全路径可访问
    tokenCookie.setMaxAge(7 * 24 * 60 * 60); // 7 天过期
    response.addCookie(tokenCookie);

    // 5. CSRF Token 通过响应头返回（不是 Cookie，JavaScript 可以读取）
    response.setHeader("X-CSRF-Token", csrfToken);

    // 6. 返回登录信息
    LoginVO loginVO = new LoginVO();
    loginVO.setUserId(user.getId());
    loginVO.setUsername(user.getUsername());
    return Result.success(loginVO);
}

/**
 * CSRF 过滤器验证 CSRF Token
 */
@WebFilter(urlPatterns = "/api/*")
public class CsrfFilter implements Filter {

    @Autowired
    private StringRedisTemplate redisTemplate;

    // 仅验证非 GET 请求
    private final Set<String> SAFE_METHODS = Set.of("GET", "HEAD", "OPTIONS");

    @Override
    public void doFilter(ServletRequest request, ServletResponse response,
                         FilterChain chain) throws IOException, ServletException {

        HttpServletRequest httpRequest = (HttpServletRequest) request;
        HttpServletResponse httpResponse = (HttpServletResponse) response;

        // GET、HEAD、OPTIONS 请求不验证 CSRF
        if (SAFE_METHODS.contains(httpRequest.getMethod())) {
            chain.doFilter(request, response);
            return;
        }

        // 从请求头获取 CSRF Token
        String csrfToken = httpRequest.getHeader("X-CSRF-Token");

        if (csrfToken == null || csrfToken.isEmpty()) {
            httpResponse.setStatus(403);
            httpResponse.setContentType("application/json;charset=UTF-8");
            httpResponse.getWriter().write("{\"code\":403,\"message\":\"CSRF token missing\"}");
            return;
        }

        // 从 Header 获取用户 ID（由认证拦截器设置）
        String userId = (String) httpRequest.getAttribute("userId");

        // 验证 CSRF Token
        String storedToken = redisTemplate.opsForValue().get("csrf:" + csrfToken);

        if (storedToken == null || !storedToken.equals(userId)) {
            httpResponse.setStatus(403);
            httpResponse.setContentType("application/json;charset=UTF-8");
            httpResponse.getWriter().write("{\"code\":403,\"message\":\"Invalid CSRF token\"}");
            return;
        }

        // 验证通过，继续处理
        chain.doFilter(request, response);
    }
}
```

```javascript
// src/utils/request.js
import axios from 'axios'

const request = axios.create({
    baseURL: '/api',
    timeout: 10000
})

// 请求拦截器：自动携带 Cookie
request.interceptors.request.use(
    config => {
        // withCredentials: true 确保浏览器发送 HTTP-only Cookie
        config.withCredentials = true

        // 从 localStorage 或页面状态获取 CSRF Token
        const csrfToken = localStorage.getItem('csrfToken')
        if (csrfToken) {
            config.headers['X-CSRF-Token'] = csrfToken
        }

        return config
    },
    error => {
        return Promise.reject(error)
    }
)

// 响应拦截器：处理 401、403 等情况
request.interceptors.response.use(
    response => {
        // 如果响应头中有新的 CSRF Token，更新存储
        const newCsrfToken = response.headers['x-csrf-token']
        if (newCsrfToken) {
            localStorage.setItem('csrfToken', newCsrfToken)
        }
        return response
    },
    error => {
        if (error.response?.status === 401) {
            // Token 过期或无效，跳转到登录页
            localStorage.removeItem('csrfToken')
            window.location.href = '/login'
        }
        return Promise.reject(error)
    }
)

export default request
```

```java
        configuration.setMaxAge(3600L);

        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/**", configuration);
        return source;
    }
```

### 3.6 安全方案对比总结

| 方案 | XSS 防护 | CSRF 防护 | 复杂度 | 推荐场景 |
|------|---------|-----------|--------|----------|
| localStorage + Header Token | ⚠️ 风险较高 | ✅ 安全 | 低 | 内部系统、风险可控 |
| HTTP-only Cookie | ✅ 安全 | ❌ 风险较高 | 中 | 需要额外 CSRF 防护 |
| HTTP-only Cookie + CSRF Token | ✅ 安全 | ✅ 安全 | 高 | 对安全性要求高的系统 |
| HTTP-only + CSRF Token + SameSite | ✅ 安全 | ✅ 安全（更强） | 高 | 现代浏览器推荐 |

### 4.1 完整的认证过滤器

```java
/**
 * JWT 认证过滤器
 * 位置：Filter（Tomcat 层），所有请求都会经过
 */
@Component
@WebFilter(urlPatterns = "/api/*")
public class JwtAuthenticationFilter implements Filter {

    @Autowired
    private JwtUtil jwtUtil;

    @Autowired
    private StringRedisTemplate redisTemplate;

    // 不需要认证的路径
    private static final Set<String> WHITE_LIST = Set.of(
        "/api/auth/login",
        "/api/auth/register",
        "/api/auth/captcha",
        "/api/auth/sendCode",
        "/api/auth/resetPassword"
    );

    @Override
    public void doFilter(ServletRequest servletRequest, ServletResponse servletResponse,
                         FilterChain chain) throws IOException, ServletException {

        HttpServletRequest request = (HttpServletRequest) servletRequest;
        HttpServletResponse response = (HttpServletResponse) servletResponse;

        String uri = request.getRequestURI();

        // 1. 白名单放行
        if (WHITE_LIST.contains(uri)) {
            chain.doFilter(request, response);
            return;
        }

        // 2. 处理预检请求
        if ("OPTIONS".equalsIgnoreCase(request.getMethod())) {
            response.setStatus(HttpServletResponse.SC_OK);
            return;
        }

        // 3. 获取 Token
        String token = extractToken(request);

        if (token == null) {
            sendUnauthorizedResponse(response, "No token provided");
            return;
        }

        // 4. 验证 Token
        try {
            Claims claims = jwtUtil.parseToken(token);

            // 检查 Token 是否被加入黑名单（用户登出时）
            String blackKey = "jwt:blacklist:" + token;
            if (Boolean.TRUE.equals(redisTemplate.hasKey(blackKey))) {
                sendUnauthorizedResponse(response, "Token has been invalidated");
                return;
            }

            // 5. 将用户信息存入请求属性，供后续使用
            request.setAttribute("userId", Long.parseLong(claims.get("userId").toString()));
            request.setAttribute("username", claims.get("username"));

            // 6. 放行
            chain.doFilter(request, response);

        } catch (ExpiredJwtException e) {
            sendUnauthorizedResponse(response, "Token expired");
        } catch (JwtException e) {
            sendUnauthorizedResponse(response, "Invalid token");
        }
    }

    /**
     * 从请求中提取 Token
     * 优先从 Header 的 Authorization 字段获取
     */
    private String extractToken(HttpServletRequest request) {
        // 方式1：从 Header 获取
        String authHeader = request.getHeader("Authorization");
        if (authHeader != null && authHeader.startsWith("Bearer ")) {
            return authHeader.substring(7);
        }

        // 方式2：从 Cookie 获取（HTTP-only Cookie）
        Cookie[] cookies = request.getCookies();
        if (cookies != null) {
            for (Cookie cookie : cookies) {
                if ("token".equals(cookie.getName())) {
                    return cookie.getValue();
                }
            }
        }

        return null;
    }

    /**
     * 发送 401 响应
     */
    private void sendUnauthorizedResponse(HttpServletResponse response, String message)
            throws IOException {
        response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
        response.setContentType("application/json;charset=UTF-8");
        response.getWriter().write("{\"code\":401,\"message\":\"" + message + "\"}");
    }
}
```

### 4.2 拦截器辅助验证

```java
/**
 * 权限拦截器
 * 位置：Interceptor（Spring MVC 层），在 Controller 执行前进行额外验证
 */
@Component
public class PermissionInterceptor implements HandlerInterceptor {

    @Autowired
    private UserService userService;

    @Override
    public boolean preHandle(HttpServletRequest request,
                            HttpServletResponse response,
                            Object handler) throws Exception {

        // 只有 Controller 方法需要权限验证
        if (!(handler instanceof HandlerMethod)) {
            return true;
        }

        HandlerMethod handlerMethod = (HandlerMethod) handler;

        // 获取方法上的 @RequiresPermission 注解
        RequiresPermission methodPermission = handlerMethod.getMethodAnnotation(RequiresPermission.class);

        // 获取类上的 @RequiresPermission 注解
        RequiresPermission classPermission =
            handlerMethod.getBeanType().getAnnotation(RequiresPermission.class);

        // 如果都没有注解，说明该方法不需要权限验证
        if (methodPermission == null && classPermission == null) {
            return true;
        }

        // 获取当前用户 ID（由认证过滤器设置）
        Long userId = (Long) request.getAttribute("userId");

        if (userId == null) {
            response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
            return false;
        }

        // 验证权限
        String[] requiredPermissions = getRequiredPermissions(methodPermission, classPermission);

        for (String permission : requiredPermissions) {
            if (!userService.hasPermission(userId, permission)) {
                response.setStatus(HttpServletResponse.SC_FORBIDDEN);
                response.setContentType("application/json;charset=UTF-8");
                response.getWriter().write("{\"code\":403,\"message\":\"Permission denied\"}");
                return false;
            }
        }

        return true;
    }

    private String[] getRequiredPermissions(RequiresPermission method, RequiresPermission classPerm) {
        // 合并类和方法上的权限要求
        List<String> permissions = new ArrayList<>();

        if (classPerm != null) {
            permissions.addAll(Arrays.asList(classPerm.value()));
        }
        if (method != null) {
            permissions.addAll(Arrays.asList(method.value()));
        }

        return permissions.toArray(new String[0]);
    }
}
```

## 五、总结

### 5.1 认证架构设计要点

1. **Filter 层**：处理全局性的认证逻辑（JWT 验证、CORS、CSRF）
2. **Interceptor 层**：处理业务层面的权限验证（日志、细粒度权限）
3. **Controller 层**：处理请求参数验证
4. **Service 层**：处理业务逻辑

### 5.2 Token 存储选择建议

| 场景 | 推荐方案 |
|------|----------|
| 练习/演示项目 | localStorage + Header Token（简单易实现） |
| 企业内部系统 | HTTP-only Cookie + CSRF Token（安全性更高） |
| 高安全性要求 | HTTP-only Cookie + CSRF Token + SameSite（最高安全） |
| 移动端 APP | localStorage + Header Token（移动端无 CSRF 风险） |

### 5.3 关键配置检查清单

- [ ] JWT Secret Key 足够长（至少 256 位）
- [ ] Token 设置合理的过期时间
- [ ] HTTP-only Cookie 设置 `HttpOnly` 和 `Secure` 属性
- [ ] 使用 HTTPS 传输
- [ ] CORS 配置正确的来源
- [ ] CSRF Token 验证非 GET 请求
- [ ] 实现 Token 黑名单机制（用户登出时）
- [ ] 敏感操作二次验证
